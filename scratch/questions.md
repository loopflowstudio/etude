# Confirmed interpretation and assumptions

- "Production Study" means the existing `GameSession.fork_study` ->
  `StudyBranch.structured_offers` -> `StudyBranch.submit` ->
  `StudyBranch.return_to_recorded` seam on current main. No test-only alternate
  branch driver is measured.
- Zero fallback applies both to source-match construction and Study execution.
  The fixed match therefore supplies a server-offer villain policy so all live
  authority fallback counters remain zero before and after the Study workload.
- Object-incarnation evidence means an authority-published `ObjectRef` is
  stable across sibling forks, the selected object performs a real zone change
  only on the child, and mutating the JSON projection cannot replace the native
  bound reference. This task does not add a new public object-inspection API.
- Performance gates bind one fixed workload, exact source closure, compiled
  managym extension, and host receipt. They are not portable hardware claims.
- Root-drift mutation is failure injection against an isolated provider built
  from a cloned retained root. There is no production root-corruption API.
- The execution receipt remains authority-private in `etude.study_branch`; the
  canonical replay and public protocol schemas remain unchanged.
